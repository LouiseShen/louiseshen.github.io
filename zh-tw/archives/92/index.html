搭建自託管評論系統——isso教程

  • 釋出時間：2020-03-29 12:14:00
  • 4506字數
  • 分類： 技術？不存在的好東西
  1. 首頁
  2. 正文

這篇文章會很長（我在寫的時候就預感到了），大致分為三大部分：

  • 安裝
  • 配置
  • 遷移

前言

為什麼搭建自託管的評論系統，對我來說，我將typecho靜態化了，所以我沒得選，只能依賴第三方服務來實現評論，我用過一段時間的gitalk，這個服務依賴於github issues功能，但是它有兩個問題：評論需要登陸，而且api.github.com在中國大陸有些地區無法訪問。對那些動態部落格使用者來說，獨立的評論系統也方便在不同的部落格系統中使用。

尋求於網際網路，isso大概是一個非常不錯的選擇：

  • 自託管
  • 支援markdown
  • 郵件回覆
  • 可從其他部落格評論系統遷移

而且它非常輕量，不需要很強效能就可以執行的很流暢。

但是它需要一臺vps，國內國外均可，且可以不用管備案限制，因為帶埠號使用https不會被攔截。

安裝

先安裝依賴（本文皆以Debian係爲例，請各位靈性代換包管理器）：

sudo apt install python-dev sqlite3 build-essential

由於isso是用python編寫的，所以從pip安裝：

pip install isso

這樣就安裝好了，安裝雖易，但是配置較繁瑣。

配置

Server配置檔案

隨便找個地方新建isso.conf，比如本文的/root/isso.conf。

寫入如下內容：

;通用配置
[general]
dbpath = /home/isso/db.db ;資料庫地址，請確保目錄存在，檔案不存在會新建
host = https://example.com ;使用isso的部落格地址 
notify = smtp ;啟用郵件通知

;服務端配置
[server]
listen = http://localhost:43956/ ;在本文使用nginx反代的方式訪問，所以只要監聽本地埠即可，用0.0.0.0表示對全網開放

;郵件通知服務
[smtp]
username = mail@example.com ;發件郵箱
password = ;你的密碼
host = smtp.example.com ;smtp伺服器
port = 465 ;smtp埠
security = ssl ;smtp加密
to = admin@example.com ;收到管理郵件的郵箱
from = "馬孔多不下雪評論通知" <mail@example.com> ;「郵件標題」 <發件郵箱>
timeout = 10

;保鏢
[guard]
enabled = true ;啟用
ratelimit = 5 ;每分鐘評論數
direct-reply = 30
reply-to-self = true ;自我回復
require-author = false ;需要使用者名稱
require-email = true ;需要電郵

;評論稽覈
[moderation]
enabled = false ;是否開啟
purge-after = 30d ;未稽覈多少天刪除

;後臺管理地址
[admin]
enabled = true ;是否啟用
password = ;設定密碼

請按照你的需要配置合理設定，更多設定可以在官網檢視

然後輸入如下命令即可執行

isso -c /root/isso.conf run

如果遇到ImportError: cannot import name SharedDataMiddleware錯誤，請輸入pip install Werkzeug==0.16.1;{1}

此時你本地迴環網路的43956埠就是這個伺服器，如果你只監聽了localhost，將無法通過外網訪問，不過不礙事，畢竟這樣沒有https，咱們通過nginx反代即可。

nginx反代

新建一個虛擬主機，配置如下：;{2,3,4}

server{
    listen 42362;

    location / {
        proxy_pass http://localhost:43956;
        gunzip on;
    }
}

server{
    listen 42361 ssl http2;
    server_name isso.example.com;

    ssl_certificate /root/key/cert.pem;
    ssl_certificate_key /root/key/key.pem;

    location / {
        proxy_pass http://localhost:42362/;
        
        sub_filter "http://localhost:43956" "https://isso.example.com:42361";
        sub_filter_once off;
        sub_filter_types *;
    }
}

請用自己的域名代換第二個server裏面的兩個isso.example.com，下面給出解釋：

本來以常規方式反代（即寫一個proxy_pass）就可以正常評論了，但是，後臺admin頁面卻無法登陸，這樣當然很不爽，所以找了找問題發現是http://localhost:43956沒有被替換，當時心裏想了想，這簡單，直接用sub_filter替換就得了，然後加上sub_filter配置。這回總行了吧，但是一登陸還是報錯，難道要帶cookie麼？想了想localhost好像沒有cookie纔對，一看發現沒有替換成功，搜尋一下才發現sub_filter不能替換gzip加密的內容，而且isso是強制啟用gzip的，淦。

那我反代不就不能用後臺了，繼續搜尋，然後發現一個奇技淫巧，真是神乎其技：

咱們反代兩次，第一次轉發後將gzip內容解壓就好了。

所以上述配置檔案中第一個server就是第一次反代，gunzip on;表示將gzip解壓。當然因為第一次僅本地訪問，所以用http就足夠了。

第二個server便是對外開放的埠。

阿里雲真好，帶埠https可以繞備案，這臺學生機終於有點用了，不再用來掛QQ了。

客戶端配置

當然僅僅是服務端配置肯定是不夠的，客戶端只要插入一段html即可：

<div id="comments">
  <section id="isso-thread">
    <script data-isso="https://isso.example.com:42361/" src="https://isso.example.com:42361/js/embed.min.js" type="text/javascript"></script>
  </section>
</div>

當然埠號是可以自己完全定義的，如果你有備案號，用443也可以。

<div>標籤只是爲了方便控制樣式，如果你部落格背景是白色的，預設樣式也足夠好看。

遷移

對於我之前使用的gitalk，並不能遷移評論，而且速度國內超慢的，完全不建議國內使用。

但是這回的isso就可以遷移評論了，當然isso是國外友人開發的，只能遷移wordpress的評論，不過這不礙事，咱們先把typecho資料匯出成wordpress的xml就好了，都不用安裝wordpress的。

這一步可以使用ByeType實現，無縫遷移。

然後得到wordpress的xml，假設叫做wp.xml並且你已經上傳到了vps上，再利用如下命令便可匯入isso資料庫：

isso -c /root/isso.conf import wp.xml

但是生活總不是一帆風順的，當你匯入過去之後你可能會發現一個小小的問題：你的原來的評論並沒有顯示出來。

看了下資料，就可以很容易的發現問題所在：

我們可以發現匯入時的連結地址和部落格的並不一致，所以我們有一個簡單的方法修改地址：

cd /home/isso/
sqlite3 db.db

此時我們應該進入到sqlite3資料庫裏面了，輸入SELECT * FROM threads;可以看到整個表，我們需要做的修改也只在這個表內。

還好/?p=數字中的這個數字和我typecho的cid是一致的，所以我們只需要將/?p=數字改為/archives/數字/即可，當然，你的連結格式不一定和我完全一樣，請靈性代換。

輸入UPDATE threads SET uri = '/archives/數字/' WHERE id = 另一個數字

注意上述id是指圖中最左邊一列的數字。

因為我只有二十條文章有評論，所以只需要一條一條改就好了。

當然如果你有上百條要改，那也不過如此(檸檬檸檬)。

另外輸入.quit可以退出sqlite資料庫。

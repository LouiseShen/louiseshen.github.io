[cdn]cloudflare大陸使用優化小技巧

  • 釋出時間：2019-08-03 10:33:00
  • 1255字數
  • 分類： 好東西
  1. 首頁
  2. 正文

近日鬼迷心竅的點上了cloudflare的橙色小云朵。（如果有注意到footer的小圖示旁會動態顯示當前cf節點就太好了）

順便優化了下，看看效果如何（17ce.com測試）：

CNAME接入

一開始我是用NS接入的，其實說實話cloudflare的DNS免費方案沒有智慧解析這一說法，所以還是推薦CNAME接入。

通過CF partner計劃即可免費CNAME接入啦，比如tloxygen。

具體接入實現還請看各面板提示。

分線路解析

國內比如dnspod就可以免費分國內外運營商解析dns，所以我們把國外訪問CNAME接入CF，國內分運營商直接A到對各運營商友好的HK/MO節點。

這類ip就不公佈了，稍微想想就可以得到的。

頁面規則

這一步很重要，因為我一開始設定錯了然後導致後臺退不出。。。

免費方案只能設定3條，不過對於typecho也是足夠使用的。

記得順序很重要，最上面的有最高的優先順序。

1 4o5.xyz/action*
  Security Level: Hign, Cache Level: Bypass, Disable Apps, Disable Performance
2 4o5.xyz/admin*
  Cache Level: Bypass, Disable Apps, Disable Performance
3 4o5.xyz/*
  Browser Cache TTL: 2 hours, Always Online: On, Cache Level: Cache Everything, Edge Cache TTL: 2 hours, Cache Deceptopn Armor: On

我看過很多部落格都沒有設定第一條，而將最後一條的Cache Level改為Standard，這樣雖然不會出現問題，但由於typecho主頁是index.php，不是靜態介面，從而導致這一步訪問會從CDN節點再向原站請求，一波完美的雲減速就這樣實現了（首位元組時間直接大於5s，不管是大陸還是美國訪問）。

另外這裏是設定過偽靜態的，沒有設定過的需要加上index.php。

請靈性代換為自己的域名，handsome主題前臺登入會失效，請使用後臺登入（我會解決的吧）

奇思妙想

既然cf和百度雲加速是聯合的，那為何不將國內直接A到su.baidu.com的ip呢？

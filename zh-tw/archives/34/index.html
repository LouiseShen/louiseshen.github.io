[nginx]nginx啟用TLS1.3

  • 釋出時間：2019-01-31 19:21:00
  • 2094字數
  • 分類： 伺服器
  1. 首頁
  2. 正文

TLS1.3即傳輸層安全協定(Transport Layer Security)的一個新版本。

發展歷史

協定釋出時間狀態
SSL1.0未公佈未公佈
SSL2.01995年已於2011年棄用
SSL3.01999年已於2015年棄用
TLS1.01999年計劃於2020年棄用
TLS1.12006年計劃於2020年棄用
TLS1.22008年
TLS1.32018年

從這裏我們可以發現上一代TLS釋出距今已有10年之久，十年時間對於網際網路的發展不可謂不大，而且TLS1.3相較於上一代有如下優勢：

  • 更快的速度：TLS1.3支援1-RTT並初步支援0-RTT（也就是握手只需要一次往返）
  • 更好的安全性：TLS1.3移除了一些不夠安全的加密演演算法

從can i use的網站上我們可以知道自Edge 79,Firefox 6, Chrome 70以及iOS safari 12.2開始，TLS1.3就被預設支援了，而且TLS版本是自動協商的（不支援就使用TLS1.2，取決於nginx的設定），所以TLS1.3的使用率只會越來越高，而且新增支援又沒什麼壞處（也許壞處就是需要放棄TLS1.1及以下支援）。

編譯更新

重寫注：其實nginx1.13+就支援TLS1.3了，我也不知道是否確實需要編譯nginx，但編譯也不費多大事對吧。

openssl

因為nginx的ssl依賴於openssl，我們使用openssl 1.1.1，這還是一個LTS版本。

$ wget https://www.openssl.org/source/openssl-1.1.1.tar.gz
$ tar -xzvf openssl-1.1.1.tar.gz

nginx

安裝編譯依賴

$ sudo apt install -y build-essential libpcre3 libpcre3-dev zlib1g-dev liblua5.1-dev libluajit-5.1-dev libgeoip-dev google-perftools libgoogle-perftools-dev

下載原始碼

重寫注：只需要版本大於1.13即可，版本可以自選，不一定要用這裏的，改版本號就行了

wget https://nginx.org/download/nginx-1.15.4.tar.gz
tar -xzvf nginx-1.15.4.tar.gz

然後用nginx -V檢視之前的編譯引數，當然你想要全新安裝可以跳過這一步。

複製下來然後在後面加上--with-openssl=/your-openssl-path/openssl-1.1.1（請靈性替換地址）

然後開始編譯（請靈性替換地址）

$ cd nginx-1.15.4
$ ./configure your-compilation-args
$ make
$ sudo make install

配置

如果不出意外的話，編譯成功後看下install到哪裏去了，然後執行編譯好的nginx。

引數設定只需要把每一個server裏面都加上TLSv1.3並且去除上古時代的TLSv1.1,1.0。

另外如果你是從老版本(1.15之前)升級上來的，你會發現ssl on這個選項已經被棄用了，用lisen 443 ssl代替即可。

驗證

在chrome瀏覽器的更多工具>開發者工具>安全選項卡檢視。

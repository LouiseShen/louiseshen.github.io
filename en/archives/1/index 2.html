How to backup the server via github

Louise
Server

Sep 12, 2020
views

I have rewritten this post because I think the origin sucked.

Backing up with a git service like github not only allows you to check the commit changes at any time for easy rollback, but also automatically!

(All commands are ran on UNIX-like systems.)

Preparation

  • github account
  • git
  • SSH key(we will generate one later)

Getting start

1.Generate a SSH key

Open the terminal and create the RSA key with the following command:

Then answer the following questions：

  1. Enter file in which to save the key
  2. Enter passphrase (empty for no passphrase)
  3. Enter same passphrase again

And then you get a key and a key.pub file.(We assume the file you saved is called key)

Open key.pub as a plain text file, copy the content of it, Then go to you GitHub，Right click your avatar -> Settings -> SSH and GPG keys -> New SSH key，paste the things you copied into Key input, you can write whatever you want in Title input.

We need a SSH key for using SSH mode while not HTTPS mode, so that we don’t need a password.

It’s very insecure to use the password while you are doing things automatically.
